## Cortex Architecture (approved v1.0)
- **Core concept:** "An agent with peripheral I/O" — one brain, channels are dumb pipes.
- **One Cortex per OpenClaw instance.** 1 OpenClaw = 1 Agent = 1 Cortex = Scaff.
- **Channels are peripherals** — webchat, WhatsApp, Telegram, Router, sub-agents, cron all feed into one unified session.
- **Message envelope:** Every message tagged with channel, sender, relationship (partner/internal/external/system), priority, replyContext.
- **`partner` = Serj only.** Everyone else classified by origin channel.
- **4 context layers:** System floor (always loaded), Foreground (active conversation, demand-based), Background (one-line summaries), Archived (not in context).
- **Strict serialization** — one message at a time, priority-ordered.
- **SQLite-backed message bus** — WAL mode, survives crashes, checkpoint after each turn.
- **Channel handoff is a non-issue** — unified session means switching channels just works.
- **Token economics & memory flow** — deferred to separate memory architecture discussion.
- Docs: `workspace/docs/cortex-architecture.md` (v1.0), `workspace/docs/cortex-implementation-tasks.md` (v1.0)
