# Cooking

**Who works here:** Scaff + Serj, Serj approve the task/idea and Scaff moves it to TO DO;

**What this is:** The kitchen. Raw ideas, architecture thoughts, problem analysis, open questions. Nothing here needs to be clean or complete.

**What belongs here:**
- Investigation reports (like `cortex-freeze-investigation`)
- Architecture explorations
- "We should probably fix X" notes
- Research on approaches

**What to do with a task here:**
- Think it through. Add notes, questions, tradeoffs.
- When the problem is understood and a solution is clear → Scaff writes a clean implementation spec and moves the file to `ToDo/`.
- Update the `status` field to `todo` and `moved_at` to today.

**Exit criteria:** The task has a clear, unambiguous implementation plan that an executor can follow without asking questions.
